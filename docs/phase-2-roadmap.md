# Phase 2 Roadmap

## Goal

Move the Constituent Tracker from a deployed MVP to a maintainable operational platform.

## Terminology

Use **constituent** terminology throughout the product and frontend code. The tool supports broader advancement audiences than donors alone, including alumni, parents, students, faculty/staff, friends, volunteers, prospects, event attendees, and donors.

Backend route names and generated API client names may still use legacy donor terminology until a later API migration. During Phase 2, prefer constituent-facing language in UI labels, components, documentation, and new frontend modules.

## Foundation

- Centralized permissions
- CI quality gates
- Monitoring and alerting
- UI pattern library
- Staging environment

## Refactoring

- Decompose large pages (starting with Constituent Lookup)
- Feature-based folder organization
- Shared API access patterns
- Shared filter and search patterns
- Migrate frontend donor naming toward constituent naming where safe

## Product Enhancements

1. Saved constituent searches
2. Communication frequency warnings
3. Duplicate constituent detection
4. Engagement scoring
5. Dashboard improvements
6. AI-assisted recommendations

## Success Criteria

- Stable production deployment
- Staging deployment workflow
- Reduced page complexity
- Faster feature delivery
- Improved user experience
- Consistent constituent-centered terminology
