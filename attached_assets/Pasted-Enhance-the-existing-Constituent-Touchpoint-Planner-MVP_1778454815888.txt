1. CAMPAIGN HEALTH CHECK
────────────────────────────────────────

Add a Campaign Health Check step before final export.

The health check should summarize whether the campaign is ready to export and identify:
- missing campaign name
- missing campaign type
- missing audience upload
- missing planned sends
- planned sends without channel
- planned sends without campaign type
- planned sends without send date
- invalid Constituent IDs
- rejected IDs not yet reviewed
- duplicate IDs removed
- extra columns ignored
- no threshold checks run
- unresolved threshold flags
- threshold flags manually overridden
- suppressions applied
- invalid suppression IDs
- invalid seed IDs
- unusual volume patterns
- same-channel/same-date warnings
- past-dated sends
- PII warnings in notes
- export counts that appear unusually low or high
- whether export will create touchpoint records that count toward future checks

The health check should categorize findings as:
- Pass
- Warning
- Error

Rules:
- Errors should block export until fixed.
- Warnings should require acknowledgement but should not necessarily block export.
- Pass items should confirm that checks are complete.
- The backend determines the health check status.
- AI may summarize the health check in plain language, but AI must not determine pass/warning/error status.

Add a clear UI section:
“Campaign Health Check”

Example AI summary:
“This campaign is ready to export. The only remaining warning is that 75 constituents exceeded the email threshold but were manually overridden.”

Add health check results to:
- Campaign detail page
- Final Export page
- Campaign activity timeline
- Export summary where relevant

Store or log health check snapshots when a campaign is exported.

Suggested database addition if needed:
campaign_health_checks
- id
- campaign_id
- status
- findings_json
- ai_summary
- created_by_user_id
- created_at
